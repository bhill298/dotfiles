# Global Agents File

## Scope and Focus for Version-Controlled Projects
- Do not make changes unrelated to the task requested. Avoid random formatting changes, refactoring, or "improvements" that aren't part of the explicit request. Ask a question first before making any changes that could be considered out of scope.
- Goal: Keep revision control commit diffs clean and focused only on changes relevant to the commit message.

## Revision Control / Git

- Unless instructed otherwise: do not commit changes to revision control without asking first. When committing changes as the user, do so without adding yourself as a co-author. Use the user's identity for commits, not the agent's.
- If new files are added that should be committed to revision control and you aren't making the commit yourself, notify the user that new files have been added and may need to be committed.

## Testing and Verification

- Always test changes if possible and applicable.
- Verify correct syntax and ensure there are no build or lint errors before considering a task complete.
- Run relevant tests, linters, type checkers, or build processes as appropriate for the codebase.
